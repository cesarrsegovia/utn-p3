Concepto y Características de JavaScript

JavaScript es un lenguaje de programación interpretado, de alto nivel, orientado a objetos basado en prototipos, y multiparadigma. Fue creado en 1995 por Brendan Eich en Netscape y es hoy el único lenguaje nativo de los navegadores web.

Interpretado / JITNo necesita compilación previa. Los motores modernos (V8, SpiderMonkey) usan compilación Just-In-Time.
Tipado dinámicoLas variables no tienen tipo fijo; el tipo se determina en tiempo de ejecución.
MultiparadigmaSoporta programación imperativa, funcional y orientada a objetos (prototipado).
Single-threadedUn solo hilo de ejecución con event loop. La asincronía se gestiona con callbacks, Promises y async/await.
Basado en ECMAScriptEl estándar que define el lenguaje. ES6 (2015) fue la actualización más importante.
UbiquitousCorre en el navegador (front-end), en el servidor (Node.js), en apps móviles (React Native) y más.

Variables — var, let, const y Tipos de Datos

Una variable es un contenedor para almacenar valores. JavaScript tiene tres formas de declararlas, con diferencias importantes de alcance (scope) y mutabilidad.

Palabra claveScopeReasignableHoistingUso recomendado
varFunciónSíSí (undefined)Código legacy, evitar
letBloqueSíNo (TDZ)Valores que cambian
constBloqueNoNo (TDZ)Valores constantes (preferido)
// ── Tipos primitivos ── let nombre = "María"; // string let edad = 25; // number let pi = 3.14; // number (float) let activo = true; // boolean let nada = null; // null let undef; // undefined let id = Symbol("id"); // symbol // ── Tipos de referencia ── const lista = [1, 2, 3]; // Array const persona = { // Object nombre: "Ana", edad: 30 }; console.log(typeof nombre); // "string" console.log(typeof lista); // "object"
// ── Diferencia de scope ── function demoScope() { var x = 1; // scope: función let y = 2; // scope: bloque const z = 3; // scope: bloque if (true) { var x = 10; // ← SOBREESCRIBE x let y = 20; // nueva y del bloque } console.log(x); // 10 ← peligro con var console.log(y); // 2 ← let correcto } // ── Coerción de tipos ── console.log("5" + 3); // "53" (concat) console.log("5" - 3); // 2 (numérico) console.log(!!0); // false (falsy)

Operadores

Los operadores permiten realizar operaciones sobre valores. JavaScript tiene una gran variedad: aritméticos, de asignación, comparación, lógicos, de cadena, ternario y más.

// ── Aritméticos ── 10 + 3 // 13 (suma) 10 - 3 // 7 (resta) 10 * 3 // 30 (mult) 10 / 3 // 3.33.. (div) 10 % 3 // 1 (módulo) 2 ** 8 // 256 (potencia) // ── Asignación ── let n = 5; n += 3; // n = 8 n -= 2; // n = 6 n *= 2; // n = 12 n ++; // n = 13 (post-incremento) ++n; // n = 14 (pre-incremento) // ── Ternario ── const msg = n > 10 ? DOM — Document Object Model

El DOM es la representación en memoria del documento HTML como un árbol de nodos. JavaScript puede leer y modificar cualquier parte de este árbol: contenido, atributos, estilos y estructura.

// ── Leer y modificar contenido ── el.textContent = "Nuevo texto"; el.innerHTML = "<b>HTML</b>"; // ── Atributos ── el.setAttribute("href", "https://..."); el.getAttribute("href"); el.removeAttribute("disabled"); // ── Clases ── el.classList.add("activo"); el.classList.remove("activo"); el.classList.toggle("visible"); el.classList.contains("activo"); // ── Estilos ── el.style.backgroundColor = "#ff0"; el.style.display = "none"; // ── Crear e insertar nodos ── const nuevo = document.createElement("li"); nuevo.textContent = "Elemento nuevo"; lista.appendChild(nuevo); lista.insertBefore(nuevo, lista.firstChild); // ── Eliminar ── el.remove(); padre.removeChild(hijo);

BOM — Browser Object Model

El BOM permite interactuar con el navegador más allá del documento. El objeto global es window, que contiene objetos como navigator, location, history, screen y métodos de tiempo.

// ── window ── window.alert("Mensaje"); window.confirm("¿Continuar?"); // bool window.prompt("Nombre:"); // string window.innerWidth; // ancho viewport window.innerHeight; // alto viewport // ── location ── location.href; // URL completa location.hostname; // dominio location.pathname; // ruta location.search; // query string ?a=1 location.reload(); // recargar página location.href = "https://..."; // ── history ── history.back(); history.forward(); history.go(-2); // 2 páginas atrás // ── navigator ── navigator.userAgent; navigator.language; navigator.onLine;
// ── Timers ── // setTimeout: ejecutar una vez const t1 = setTimeout(() => { console.log("Pasaron 2 segundos"); }, 2000); clearTimeout(t1); // cancelar // setInterval: repetir cada N ms let count = 0; const t2 = setInterval(() => { count++; if (count >= 5) clearInterval(t2); }, 1000); // requestAnimationFrame (animaciones) function animar(ts) { // lógica de animación requestAnimationFrame(animar); }

Eventos

Los eventos permiten responder a interacciones del usuario o del navegador. Se registran con addEventListener y reciben un objeto Event con información del suceso.

// ── Formas de registrar eventos ── // 1. HTML inline (evitar) // <button onclick="fn()"> // 2. Propiedad (solo 1 handler) btn.onclick = function() { /* ... */ }; // 3. addEventListener (recomendado) btn.addEventListener("click", handler); btn.removeEventListener("click", handler); // ── El objeto Event ── function handler(e) { e.type; // "click" e.target; // elemento que disparó e.currentTarget; // elemento con listener e.preventDefault(); // cancela default e.stopPropagation(); // detiene bubbling e.clientX; e.clientY; // posición del mouse e.key; // tecla presionada } // ── Event Delegation ── lista.addEventListener("click", e => { if (e.target.matches("li")) { console.log(e.target.textContent); } });

jQuery — Selectores, Métodos y Eventos

jQuery es una biblioteca que simplifica la manipulación del DOM, el manejo de eventos y las peticiones AJAX. Se importa con <script src="jquery.min.js"> y se accede con $ o jQuery.

// ── Selectores jQuery ── $("#id") // por ID $(".clase") // por clase $("div") // por tag $("ul li:first") // CSS + pseudo $("input[type=text]") // por atributo // ── DOM ── $("#p").text("Nuevo"); $("#p").html("<b>Bold</b>"); $("#p").val(); // input value $("#p").attr("href"); // attr $("#p").css("color", "red"); // ── Clases ── $(".item").addClass("activo"); $(".item").removeClass("activo"); $(".item").toggleClass("activo"); // ── Efectos ── $("#caja").hide(); $("#caja").show(); $("#caja").toggle(); $("#caja").fadeIn(400); $("#caja").slideDown(300); // ── Chaining ── $("#el") .addClass("activo") .css("color", "blue") .text("Modificado");
// ── Eventos jQuery ── $("#btn").on("click", function() { $(this).toggleClass("activo"); }); // Shorthand $("#btn").click(handler); $("input").keyup(handler); // Delegación de eventos $("#lista").on("click", "li", function() { $(this).toggleClass("selected"); }); // AJAX con jQuery $.ajax({ url: "/api/data", method: "GET", success: data => console.log(data), error: err => console.error(err) }); // ── Document Ready ── $(function() { // se ejecuta cuando el DOM está listo console.log("DOM listo"); });

Funciones — Declaración, Expresión, Arrow, Callbacks, Closures

Las funciones son ciudadanos de primera clase en JavaScript: se pueden asignar a variables, pasar como argumentos y retornar desde otras funciones.

// ── Declaración ── function sumar(a, b) { return a + b; } // ── Expresión ── const restar = function(a, b) { return a - b; }; // ── Arrow function ── const mult = (a, b) => a * b; const doble = n => n * 2; // ── Parámetros por defecto ── function saludar(nombre = "visitante") { return `Hola, ${nombre}!`; } // ── Rest params ── function total(...nums) { return nums.reduce((a, b) => a + b, 0); } total(1, 2, 3, 4); // 10
// ── Callback ── function operar(a, b, cb) { return cb(a, b); } operar(4, 5, mult); // 20 // ── Closure ── function contador(inicio = 0) { let n = inicio; return { incrementar: () => ++n, valor: () => n, reset: () => { n = inicio; } }; } const c = contador(10); c.incrementar(); // 11 c.valor(); // 11 // ── IIFE ── (function() { console.log("Ejecuta inmediatamente"); })(); // ── Async / Await ── async function fetchData(url) { const res = await fetch(url); const json = await res.json(); return json; }

Manejo de Excepciones — try / catch / finally / throw

El manejo de errores permite que el programa responda de forma controlada ante situaciones inesperadas, en lugar de colapsar.

// ── try / catch / finally ── try { // código que podría fallar const data = JSON.parse("json_inválido"); } catch (err) { console.error(err.name); // SyntaxError console.error(err.message); // descripción } finally { // siempre se ejecuta console.log("Bloque finally"); } // ── throw personalizado ── function dividir(a, b) { if (b === 0) throw new Error("División por cero"); return a / b; } // ── Clase de error personalizada ── class ValidationError extends Error { constructor(campo, msg) { super(msg); this.name = "ValidationError"; this.campo = campo; } }
// ── Tipos de error nativos ── // SyntaxError — JSON.parse("x") // TypeError — null.propiedad // ReferenceError— varNoDeclarada // RangeError — new Array(-1) // URIError — decodeURI("%") // ── Catch diferenciado ── try { procesarDatos(); } catch (err) { if (err instanceof ValidationError) { mostrarError(err.campo, err.message); } else if (err instanceof TypeError) { console.error("Tipo inválido", err); } else { throw err; // relanzar si no lo manejamos } } // ── Async error handling ── async function cargar() { try { const r = await fetch("/api"); if (!r.ok) throw new Error(r.status); return await r.json(); } catch(e) { console.error("Fetch error:", e); } }
